Server Tuning

Global knobs for background workers, retention, and reporting cadence.

Repository statistics

How often the CLI runs restic stats per job. Default 86400 (24h). Set per-job overrides on the job edit page.

Data retention

Full detail kept this long. Older reports get heartbeats pruned and rolled into daily aggregates.
Beyond this age, only daily aggregates survive.
0 = keep forever. Applies to both server-originated and node-originated audit rows.

Offline detection

Nodes not checking in for this long are marked offline.
How often the worker scans for offline nodes.
Aggressive "node went silent" detector — fires a node_silent audit-log warn if a node misses its heartbeat for this long. Set lower than the offline threshold so attacker-stopped daemons get flagged within minutes. 0 disables. Default 7.

Alerts

Used when users silence an alert without picking a duration.

Terminal Session Recording

Every SSH session opened from the dashboard is recorded end-to-end (keystrokes + output) in asciinema v2 format. Superadmins can replay from the audit log. Warning: anything typed or printed is captured including pasted secrets.

A red banner appears on the terminal page while recording is active.
0 = keep forever. Older .cast files are pruned from disk hourly.

Anomaly Tuning

Controls the global Security — Anomalies page: thresholds for what fires, plus how long acknowledged rows stay on the live view before moving to the archive.

After this many days, acknowledged anomalies disappear from /anomalies but remain forever on /anomalies/archive. Default 30.

Anomaly Thresholds

Triggers when a backup run shows unexpected drops vs. the previous run — possible signs of source deletion, ransomware, or repo tampering. Each anomaly is logged for forensics.

Alert when the restic repo loses more than this many snapshots between two reports. Default 1.
